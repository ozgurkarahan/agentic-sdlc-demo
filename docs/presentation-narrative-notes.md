# Presentation Narrative Notes (Post-Meeting)

## What to improve

- Structure the story through clear personas:
  - The person setting up the harness
  - The person templatizing it in GitHub
  - The person using it inside a project
  - The PM receiving team recap outputs to create inputs and working units for project management tools (for example, Jira)
  - The person validating issues so work can run and be implemented
  - Show the full workflow end-to-end through these personas

- Start the presentation with:
  1. Harness
  2. Workflow
  3. Agent skills
  4. Human-in-the-loop points
  - Avoid introducing other tools/products before this foundation.

- Then transition to:
  - “Here is how I used it in an example”
  - Live demo walkthrough

- At the beginning, also cover:
  - Framework/patterns
  - Flexibility and customization options
